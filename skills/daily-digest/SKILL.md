# Daily Digest Skill

Compiles a daily summary of newly discovered leads and email draft previews, then delivers it
to the CEO via their preferred channel(s): Email, Telegram, or WhatsApp.

---

## Digest Content Structure

The daily digest contains:

```
📋 CEO ASSISTANT — DAILY LEAD DIGEST
📅 [Date] | [N] New Leads Found

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

LEAD SUMMARY
────────────
[For each new lead:]

🏛️ [Municipality Name] — [Region]
👤 [Contact Name], [Role]
📧 [Email]
📞 [Phone or "Not found"]
🌐 [Source URL]

✉️ EMAIL DRAFT PREVIEW:
Subject: [Draft subject line]
---
[First 3–4 lines of draft email...]
[...] (full draft in Gmail/Sheets)
---

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 STATS
• New leads today: [N]
• Total leads in database: [N]
• Leads awaiting review: [N]
• Google Sheets: [link]
• Gmail Drafts: [link to Gmail drafts folder]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[Generated by CEO Assistant at 07:00 CET]
```

## Delivery Channels

### 1. Email (via Gmail MCP)

Send digest as a formatted HTML email to the CEO's address.

**Gmail MCP**: `https://gmailmcp.googleapis.com/mcp/v1`

Format:
- Subject: `📋 CEO Digest — [N] New Leads — [Date]`
- Body: HTML-formatted digest with clickable links to Sheets and Gmail drafts
- To: CEO email address (ask once, then reuse)

### 2. Telegram Bot

Send digest as a Telegram message via Bot API.

Required config (ask user once):
- Telegram Bot Token
- CEO's Telegram Chat ID

API call:
```
POST https://api.telegram.org/bot{TOKEN}/sendMessage
{
  "chat_id": "{CHAT_ID}",
  "text": "[digest text]",
  "parse_mode": "Markdown"
}
```

Telegram message format: shortened version (max 4096 chars), with link to full Sheets view.

### 3. WhatsApp (via Twilio)

Send digest via WhatsApp using Twilio API.

Required config:
- Twilio Account SID
- Twilio Auth Token
- Twilio WhatsApp number
- CEO WhatsApp number

Message: shortened digest (first 2 leads + stats + Sheets link), full details in email.

## Workflow

### Step 1 — Collect Today's Data
Pull new leads from one of:
- Conversation context (output from lead-researcher / gmail-lead-drafter)
- Google Sheets (filter Status=`New` and Date Found=today)

### Step 2 — Compile Digest
Build the digest text using the template above.
- Include all new leads found today
- Include first 3–4 lines of each email draft preview
- Include stats block with totals

### Step 3 — Deliver via Configured Channels
Send to all configured channels. Report delivery status per channel.

### Step 4 — Confirm Delivery
```
📬 Daily Digest Sent

✅ Email → [CEO email]
✅ Telegram → CEO chat
✅ WhatsApp → +420...

[N] leads included in today's digest.
Next digest scheduled: tomorrow at 07:00 CET.
```

## Scheduling Note

The daily digest is designed to run at **07:00 CET** as the final step of the daily workflow:

```
07:00 — Lead Scout runs
07:05 — Data Enricher + Deduplication
07:10 — Email Drafter generates drafts
07:15 — Gmail drafts saved
07:20 — Sheets updated
07:25 — Daily Digest sent to CEO ← this skill
```

Scheduling itself requires an external trigger (cron job, n8n, Make.com, or cloud scheduler).
This skill handles the compilation and delivery step only.

---

## Error Handling

- **No new leads today**: Send digest with "No new leads found today" message — still send stats
- **Gmail MCP unavailable**: Skip email, deliver via Telegram/WhatsApp only, flag in report
- **Telegram error**: Log error, continue with other channels
- **WhatsApp error**: Log error, continue with other channels
- **All channels fail**: Display digest in conversation for manual forwarding
